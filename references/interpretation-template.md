# 分层解读模板

按以下模板组织回答，避免跨层误用结论。

1. 输入确认
- 出生信息（公历/农历、日期、时辰、性别、出生地）
- 出生信息确认状态（必须为已确认）
- 当前参考时区
- 参考日期（公历 + 农历）
- 数据口径声明：默认使用 `byRole`（宫位角色映射）

2. 本命盘（`natalSummary` + `currentDetailed.palaces[].natal`）
- 命宫/身宫核心结构
- 主要星曜与性格、长期倾向

3. 流年（本命 + yearly 层数据）
- 年度主线、机会、风险

4. 流月（本命 + yearly + monthly 层数据）
- 本月节奏与重点事项

5. 流日（本命 + yearly + monthly + daily 层数据）
- 当日建议、执行优先级

6. 未来辅助（`futureDetailed[]`）
- 按日期对比趋势变化
- 标注关键窗口（上升/回撤/观望）

7. 限制与假设
- 缺失字段
- 未做真太阳时修正
- 结论只用于文化研究与娱乐参考

补充：
- 当用户要求"尽量详细"时，优先遍历 `currentDetailed.palaces[]` 输出逐宫信息，并附上 `tags`（本命/大限/流年/流月/流日/流时）。
- 当前 skill 固定输出为 full：默认输出 12 宫完整明细，不降级为简版。
- 仅在开发调试时使用索引口径（`query.debug.includeIndexMapping=true`）。
