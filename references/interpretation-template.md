# 分层解读模板

按以下模板组织回答，避免跨层误用结论。

1. 输入确认
- 出生信息（公历/农历、日期、时辰、性别、出生地）
- 出生信息确认状态（必须为已确认）
- 当前参考时区
- 参考日期（公历 + 农历）
- 数据口径声明：默认使用 `byRole`（宫位角色映射）

2. 本命盘（`natalSummary` + `currentDetailed.palaces[].natal`）
- 命宫/身宫核心结构
- 主要星曜与性格、长期倾向

3. 流年（本命 + yearly 层数据）
- 年度主线、机会、风险

4. 流月（本命 + yearly + monthly 层数据）
- 本月节奏与重点事项

5. 流日（本命 + yearly + monthly + daily 层数据）
- 当日建议、执行优先级

6. 未来辅助（`futureDetailed[]`）
- 按日期对比趋势变化
- 标注关键窗口（上升/回撤/观望）

7. 限制与假设
- 缺失字段
- 未做真太阳时修正
- 结论只用于文化研究与娱乐参考

补充：
- 禁止仅输出"星盘 + 一句话评价"。除非用户明确要求简版，默认输出 full 深度结果。
- 默认遍历 `currentDetailed.palaces[]` 输出 12 宫完整明细，并附上 `tags`（本命/大限/流年/流月/流日/流时）。
- 每条关键结论必须配证据链：`证据（宫位/星曜组合/四化或tags） -> 领域落点 -> 推断结论`。
- 关键判断需区分层级来源（本命、流年、流月、流日），禁止跨层归因。
- 中州派解读至少引用 2 个 `wang-references/` 文件观点（通常为命宫主星文件 + 格局或总论）。
- 仅在开发调试时使用索引口径（`query.debug.includeIndexMapping=true`）。
- 若平台有长度限制，按"第1部分/第2部分"分批连续输出，不降级为极简摘要。
